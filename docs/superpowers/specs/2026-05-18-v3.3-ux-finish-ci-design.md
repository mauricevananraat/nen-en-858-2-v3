## 2. Items (definitief)

### 2.1 Pagina-nummering — voorpagina geen footer

**Bestand:** `js/pdf-builder.js`

**Probleem:** voorpagina toont "Pagina 1 van 12" terwijl voorpagina visueel een dedicated rapport-cover is (geen nummering passend).

**Beslissing:** voorpagina krijgt geen footer. Sectie-pagina's (vanaf pagina 2 in PDF) tellen 1 t/m N-1.

**Implementatie:** in `buildDocDefinition`, footer-callback returnt `null` voor pagina 1; voor andere pagina's `currentPage - 1` van totaal `pageCount - 1`.

### 2.2 §6 Inwendige controle — thumbnails 110px

**Bestand:** `js/pdf-builder.js`

**Probleem:** §6 (5 sub-blokken: wanden, schotten, coalescentie, afsluiter, naden, elk met 1 foto) valt over 2 pagina's met 140×140 thumbnails. Pagina 9 toont alleen 1 foto-blok.

**Beslissing:** alleen §6-photoBlocks krijgen 110×110 thumbnails. Andere secties (1-5, 7, 8) behouden 140×140.

**Implementatie:** `photoBlock(photos, title)` wordt `photoBlock(photos, title, size = 140)`. In `buildSection6Inwendig`-functie: alle photoBlock-calls geven `size: 110` als 3e argument. `columns` widthwaarde wordt `size + 10` (kolomruimte iets ruimer dan thumbnail).

Als content nog steeds niet past op 1 pagina: pdfMake breekt natuurlijk tussen sub-blokken (unbreakable photoBlock voorkomt halve foto's). Geen verdere fallback nodig.

### 2.3 Spinner safety-net — 10s → 20s

**Bestand:** `js/main.js`

**Probleem:** pdfMake.download-callback firet soms niet betrouwbaar op trage devices met grote rapporten (28+ foto's). 10s is dan te kort, spinner verdwijnt vóór download triggert.

**Beslissing:** safety-net naar 20s. Constante bovenaan main.js zodat het 1 plek is om te tunen.

**Implementatie:**
```js
const PDF_SPINNER_TIMEOUT_MS = 20000;
// ...
setTimeout(hideSpinner, PDF_SPINNER_TIMEOUT_MS);
```

NOTE: dit raakt ook item 4 — pre-existing magic number `10000` wordt nu constante. Andere magic numbers (toast: 4000, sync-ui: 1500) blijven onveranderd per Maurice's keuze.

### 2.5 GitHub Actions CI

**Bestand:** `.github/workflows/test.yml` (nieuw)

**Probleem:** geen automatische test-guard bij commits. Risico: regressie sluipt stilzwijgend live.

**Beslissing:** workflow draait `npm test` bij elke push naar `main` én elke pull request. Tests moeten groen voor merge.

**Implementatie:**
```yaml
name: Tests
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm test
```

Verwacht: groene badge bij eerste push. Maurice ontvangt automatisch e-mail bij faal (GitHub-default).

### 2.6 Skipped sync-ui test verwijderen — cleanup

**Bestand:** `tests/sync-ui.test.js` (regels 161-182)

**Probleem:** `it.skip('vraagt mode via confirm bij bestand-selectie', ...)` test functionaliteit die in v3.1 is vervangen door `openImportModeModal`. Het bevatte een tautologie-assertie (`addEventListener` is altijd truthy) en werd in fase 5 al gemarkeerd als "no-op" door code-reviewer.

**Beslissing:** verwijder het hele `describe('bindSyncButtons — import-mode dialog', ...)` blok rond regel 161. Het sub-test `is idempotent — 2x bindSyncButtons` (regel 184) blijft behouden — verplaats zo nodig naar een ander describe.

**Resultaat:** 0 skipped tests. Coverage van mode-modal-flow zit volledig in `tests/import-mode-modal.test.js`.

## 7. Risico's en mitigaties

| Risico | Mitigatie |
|--------|-----------|
| §6 110×110 thumbnails te klein voor visuele inspectie van scheuren | Test in echte 5-jaarlijks PDF — zo niet, fallback naar 120×120 of accepteer 2 pagina's |
| Footer-callback met `currentPage - 1` faalt als pageCount = 1 (alleen voorpagina) | Edge-case onmogelijk in dit project: er is altijd minstens 1 sectie na voorpagina |
| GitHub Actions `npm ci` faalt door pakket-lock-issue | Lokale `npm ci` uitvoeren vóór commit om dit te vangen |
| Skipped-test verwijderen breekt onverwacht andere test | `npm test` bevestigt vóór commit |
| Pages-cache toont oude PDF na fix | Bestaande update-toast (v3.2) informeert gebruiker; CACHE_VERSION-bump in sw.js |
